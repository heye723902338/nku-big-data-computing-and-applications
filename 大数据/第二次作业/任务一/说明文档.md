# 推荐系统任务一优化文档：项目要求与往届性能参考

整理日期：2026-06-11

## 1. 项目任务要求

### 1.1 任务目标

本次任务一的核心目标是：

> 根据 `train.txt` 中已有的用户-物品评分记录，预测 `test.txt` 中每个用户-物品对 `(u, i)` 的未知评分。

也就是说，我们需要训练一个推荐系统评分预测模型，并输出与 `ResultForm.txt` 一致格式的预测结果文件。

### 1.2 可使用方法

作业要求允许使用课程中学过的推荐算法，也允许使用其他资源中学习到的算法。因此可选方法包括但不限于：

- 全局均值、用户均值、物品均值
- 用户/物品偏置模型
- UserCF、ItemCF、KNN 协同过滤
- MF、SVD、PMF、SVD++
- NCF、HybridCF、浅层神经网络
- 多模型融合、加权 ensemble、stacking

但需要注意：报告和代码必须由本小组独立完成，不能直接复制往届代码或报告。

### 1.3 报告要求

项目报告至少应包含以下内容：

1. 数据集基本统计信息：用户数、物品数、评分数、评分分布、稀疏度等。
2. 算法细节：模型原理、目标函数、训练流程、参数设置。
3. 实验结果：RMSE、训练时间、预测时间、空间/内存消耗。
4. 理论分析或实验分析：为什么某些算法有效，为什么某些算法表现较差。

## 2. 数据与提交格式

### 2.1 `train.txt`

格式如下：

```text
<user id>|<numbers of rating items>
<item id>   <score>
```

说明：

- 每个用户块第一行是 `用户ID|评分数量`。
- 接下来若干行是该用户对物品的评分。
- 评分范围为 10 到 100。

### 2.2 `test.txt`

格式如下：

```text
<user id>|<numbers of rating items>
<item id>
```

说明：

- 每个用户块第一行是 `用户ID|待预测数量`。
- 接下来若干行只有物品 ID，需要我们预测评分。

### 2.3 结果文件

结果文件应保持 `test.txt` 的用户顺序和物品顺序，并补上预测分数：

```text
<user id>|<numbers of rating items>
<item id>   <score>
```

重要检查点：

- 不要打乱用户顺序。
- 不要打乱每个用户内部的物品顺序。
- 不要漏掉测试集中任何一个 `(user, item)`。
- 不要错误假设每个用户固定只有 6 个或 20 个物品。
- 预测值建议保留连续浮点数，除非提交系统明确要求整数。

## 6. 对我们优化方向的初步判断

往届资料不是天花板。部分代码存在验证不严谨、冷启动粗糙、训练 RMSE 与验证 RMSE 混用、预测强行 round、输出格式假设错误等问题。

结合当前数据实验和往届资料，建议将目标分为三档：

| 档位 | 目标 RMSE | 说明 |
|---|---:|---|
| 基础合格 | 17.5-18.5 | bias、均值融合、简单 MF 可以达到。 |
| 稳健优秀 | 16.8-17.2 | 正则化 MF/SVD + bias + 基础冷启动。 |
| 冲刺目标 | 16.3-16.6 | 需要多模型 ensemble、残差 KNN、OOF 验证、分层冷启动和校准。 |
| 极限尝试 | 16.0-16.3 | 需要进一步 stacking、多随机种子、多参数模型融合，且不保证稳定。 |

## 7. 后续建议尝试的三条优化路线

旧版 baseline 应该保留，作为后续所有增强模型的对照组。接下来建议按“稳定增强 -> 主力提升 -> 冲刺融合”的顺序推进，避免一开始直接上复杂模型导致问题难以定位。

| 方案 | 模型组合 | 适合程度 | 预期效果 |
|---|---|---|---|
| 方案一：稳健增强版 | `Tuned BaselineBiasModel + Shrink Fallback` | 最容易实现 | 小幅提升，稳定 |
| 方案二：推荐主力版 | `ALS Bias + Residual ItemKNN + Shrink Fallback` | 最推荐 | 明显提升，性价比最高 |
| 方案三：冲刺融合版 | `ALS Bias + Residual ItemKNN + Tuned MF + Ensemble` | 最强但更复杂 | 有机会进一步冲高分 |

推荐实验对比顺序：

```text
旧版 baseline
方案一：稳健增强版
方案二：推荐主力版
方案三：冲刺融合版
```

每一步都要在同一套验证集切分上记录 RMSE、训练时间、预测时间、冷启动分层表现，避免只看单次结果。

## 8. 方案一：稳健增强版

### 8.1 核心思路

方案一保留当前 baseline 的整体思路，但做三类低风险增强：

```text
1. 调 baseline 的 epochs、lr、reg
2. cold user / cold item 使用 shrink mean
3. 输出小数，不默认四舍五入
```

该方案适合作为“旧版增强模型”。实现快，风险低，不会大幅改变现有代码结构。

### 8.2 模型形式

推荐使用如下预测形式：

```text
pred(u, i) = global_mean + user_bias[u] + item_bias[i]
```

其中 user bias 和 item bias 不建议直接使用简单均值差，而应加入正则化：

```text
user_bias[u] = sum(r_ui - global_mean - item_bias[i]) / (lambda_user + count_user[u])
item_bias[i] = sum(r_ui - global_mean - user_bias[u]) / (lambda_item + count_item[i])
```

通过 ALS 方式交替更新若干轮，得到更稳定的偏置。

### 8.3 Shrink Fallback

冷启动不能直接全部返回全局均值，建议按场景处理：

| 场景 | 推荐预测 |
|---|---|
| warm user + warm item | `global_mean + user_bias + item_bias` |
| warm user + cold item | `global_mean + user_bias`，必要时向全局均值 shrink |
| cold user + warm item | `global_mean + item_bias`，必要时向全局均值 shrink |
| cold user + cold item | `global_mean` |
| 低频用户/低频物品 | 使用 shrink 后的 user/item mean |

Shrink mean 示例：

```text
shrunk_user_mean[u] = (count_user[u] * user_mean[u] + alpha * global_mean) / (count_user[u] + alpha)
shrunk_item_mean[i] = (count_item[i] * item_mean[i] + beta * global_mean) / (count_item[i] + beta)
```

其中 `alpha`、`beta` 需要在验证集上调参。

可以把这一段作为实现提示发给负责代码的同学：

```text

要求：

3. 使用 ALS 交替学习 user_bias 和 item_bias。
4. 支持参数 lambda_user、lambda_item、n_iter。

```

### 8.5 预期收益

该方案预期不会产生特别大的提升，但能把当前 baseline 做扎实，适合作为后续方案二和方案三的对照基线。

## 9. 方案二：推荐主力版

### 9.1 核心思路

方案二是最推荐重点实现的路线：

```text
最终预测 = ALS Bias 预测 + Residual ItemKNN 残差修正
```

含义：

```text
先用 ALS Bias 学全局均值、用户偏置、物品偏置
再用 ItemKNN 学相似物品之间还没被 baseline 解释掉的残差
```

这个方案比单纯 baseline 更能捕捉物品局部相似性，适合当前数据：用户数较少，物品数较多，很多物品评分稀疏，直接做复杂深度模型不一定稳定，而残差 ItemKNN 可以补充矩阵分解或 bias 没解释掉的局部关系。

### 9.2 残差定义

先训练 ALS Bias：

```text
base_pred(u, i) = global_mean + user_bias[u] + item_bias[i]
```

然后对每条训练评分计算残差：

```text
residual(u, i) = rating(u, i) - base_pred(u, i)
```

ItemKNN 不直接对原始评分做相似度，而是对残差做相似度，这样可以避免用户打分习惯和物品整体热度干扰相似度。

### 9.3 ItemKNN 残差修正

对目标 `(u, i)`，找到用户 `u` 已经评分过的物品集合 `N(u)`，计算目标物品 `i` 与这些物品 `j` 的相似度，然后加权残差：

```text
residual_hat(u, i)
= sum(sim(i, j) * residual(u, j)) / sum(abs(sim(i, j)))
```

最终预测：

```text
pred(u, i) = base_pred(u, i) + residual_hat(u, i)
```

相似度建议使用 shrink 后的 cosine / Pearson：

```text
sim_shrink(i, j) = sim_raw(i, j) * common_count(i, j) / (common_count(i, j) + shrink)
```

这样可以避免两个物品只被极少用户共同评分时产生虚高相似度。

### 9.4 需要调的参数

| 参数 | 含义 | 建议范围 |
|---|---|---|
| `lambda_user` | 用户偏置正则 | 5, 8, 10, 15, 20 |
| `lambda_item` | 物品偏置正则 | 5, 8, 10, 15, 20 |
| `bias_iter` | ALS bias 迭代次数 | 20, 30, 40 |
| `top_k` | ItemKNN 使用的相似物品数量 | 20, 50, 100 |
| `min_common` | 两个物品最少共同评分用户数 | 2, 3, 5 |
| `shrink` | 相似度收缩系数 | 5, 10, 20, 30 |
| `residual_weight` | 残差修正权重 | 0.3, 0.5, 0.7, 1.0 |

### 9.5 实现提示词

可以把这一段作为实现提示发给负责方案二的同学：

```text
请实现 ALS Bias + Residual ItemKNN 模型。

要求：
1. 先实现 ALS Bias：
   pred_base = global_mean + user_bias + item_bias。
2. 在训练集上计算 residual = rating - pred_base。
3. 构建 item -> {user: residual} 和 user -> {item: residual} 两个索引。
4. 基于残差计算 item-item 相似度。
5. 相似度必须加入 shrink：
   sim = raw_cosine * common_count / (common_count + shrink)。
6. 预测 warm pair 时：
   pred = ALS_Bias_pred + residual_weight * ItemKNN_residual。
7. 如果找不到可用邻居，则退回 ALS_Bias_pred。
8. cold user / cold item / both cold 使用 shrink fallback。
9. 参数 top_k、shrink、min_common、residual_weight 必须通过验证集调。
10. 输出连续预测值并 clip 到 [10, 100]。
11. 记录整体 RMSE 和 warm/cold 分层 RMSE。
```

### 9.6 预期收益

该方案是性价比最高的主力模型。它比单纯 baseline 更强，又比复杂神经网络更可控。根据当前数据的内部实验，`ALS Bias + Residual ItemKNN` 方向具备明显提升空间，适合作为最终提交的核心候选方案。

## 10. 方案三：冲刺融合版

### 10.1 核心思路

方案三在方案二基础上，再加入一个调好参数的 MF：

```text
最终预测 =
0.55 * ALS_ItemKNN
+ 0.25 * ALS_Bias
+ 0.10 * Tuned_Baseline
+ 0.10 * Tuned_MF
```

注意：上面的权重只是初始建议，不应拍脑袋固定。最好使用验证集或 OOF 预测来调权重。

### 10.2 为什么加入 MF

Residual ItemKNN 更擅长利用局部物品相似性，但对长尾物品和稀疏邻居可能不稳定。MF 可以学习低维潜在因子，对 warm pair 有补充价值。

但 MF 也有风险：

- 默认参数容易过拟合。
- 维度太大可能拖后腿。
- 学习率太大容易震荡。
- 没有 early stopping 会越训越差。
- 如果没有 bias 项，效果通常不稳定。

因此 MF 必须是 tuned MF，而不是直接使用当前默认参数。

### 10.3 MF 建议形式

推荐形式：

```text
pred_mf(u, i)
= global_mean + user_bias[u] + item_bias[i] + dot(P[u], Q[i])
```

训练目标：

```text
min sum((r_ui - pred_mf(u, i))^2)
    + reg_bias * (user_bias^2 + item_bias^2)
    + reg_factor * (||P[u]||^2 + ||Q[i]||^2)
```

参数建议：

| 参数 | 建议 |
|---|---|
| 隐向量维度 | 20, 40, 60, 80 |
| 正则强度 | 偏强正则，优先避免过拟合 |
| 训练方式 | SGD 或 ALS |
| early stopping | 必须使用验证集 |
| 输出 | 连续分数，clip 到 [10, 100] |

### 10.4 Ensemble 权重调优

建议不要只调一个最终 RMSE，而要保存每个模型在验证集上的预测：

```text
pred_bias
pred_itemknn
pred_mf
pred_baseline
```

然后在验证集上搜索融合权重：

```text
pred_final =
w1 * pred_itemknn
+ w2 * pred_bias
+ w3 * pred_baseline
+ w4 * pred_mf
```

约束：

```text
w1 + w2 + w3 + w4 = 1
w1, w2, w3, w4 >= 0
```

如果时间允许，建议做 OOF stacking：

```text
1. 划分 K 折验证集。
2. 每一折训练所有基础模型。
3. 保存该折验证集预测，形成 OOF 预测矩阵。
4. 用 OOF 预测学习融合权重。
5. 全量训练基础模型。
6. 使用学到的权重融合 test 预测。
```


### 10.6 预期收益

方案三是最强但也最复杂的路线。如果实现和验证严谨，有机会进一步冲高分。但如果 MF 参数没有调好，或融合权重没有用验证集学习，可能反而拖累方案二。

因此建议先把方案二做到稳定，再上方案三。

### 13.3 训练集评分分布图

建议标题：

```text

```

作用：

- 如果评分集中在 60、80、100，可以说明模型需要处理偏高评分分布。
- 为后续预测分布校准提供依据。

制作建议：

- 使用柱状图。
- 可以在图下方补一句分析：

```text
训练集评分主要集中在中高分区间，说明用户评分存在整体偏高倾向，因此模型需要显式建模全局均值和用户/物品偏置。
```

### 13.4 用户/物品评分次数长尾分布图

建议标题：

```text
用户与物品评分次数分布
```

可以做两张子图：

```text
左图：每个用户的评分数量分布
右图：每个物品的被评分数量分布
```

作用：

- 展示数据稀疏性和长尾现象。
- 说明很多物品评分次数很少，直接用 item mean 或 KNN 容易不稳定。
- 为 shrink mean、正则化 bias、cold fallback 提供动机。

制作建议：

- 建议使用 log 坐标或横向箱线图，否则长尾会把主体压扁。
- 如果做直方图，可以把横轴设为评分次数区间。
- 图下方可以补一句：

```text
物品侧长尾明显，低频物品的均值估计方差较大，因此需要通过 shrink 和正则化降低过拟合风险。
```

建议标题：

### 13.7 训练过程 RMSE 曲线

建议标题：

```text

```

建议内容：

作用：

- 判断是否过拟合。

制作建议：

### 13.8 预测分布 vs 真实分布图

建议标题：

```text
验证集真实评分与预测评分分布对比
```

建议内容：

- 在验证集上画真实评分分布。
- 同时画模型预测评分分布。

作用：

- 检查模型是否预测过于集中。
- 检查是否整体偏高或偏低。
- 判断是否需要预测分布校准。

制作建议：

- 可以使用叠加直方图或 KDE 曲线。
- 如果预测分布明显比真实分布窄，说明模型可能过度回归均值。
- 如果预测均值明显偏离真实均值，说明需要校准。
